# Self-Hosted LangFuse Deployment Guide

Complete guide for deploying LangFuse on your own infrastructure and integrating with GraphRAG.

---

## Quick Start: Docker Compose (5 minutes)

### Simplest Setup

```bash
# 1. Create deployment directory
mkdir langfuse && cd langfuse

# 2. Create docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  langfuse:
    image: langfuse/langfuse:latest
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://postgres:password@postgres:5432/langfuse
      NEXTAUTH_SECRET: your-secret-key-here
      NEXTAUTH_URL: http://localhost:3000
    depends_on:
      - postgres

  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
      POSTGRES_DB: langfuse
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  postgres_data:
EOF

# 3. Start services
docker-compose up -d

# 4. Access at http://localhost:3000
```

## Production Setup: Docker Compose with Persistence

```bash
# Create secure secrets
NEXTAUTH_SECRET=$(openssl rand -base64 32)
DB_PASSWORD=$(openssl rand -base64 16)

cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  langfuse:
    image: langfuse/langfuse:latest
    container_name: langfuse-app
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: production
      DATABASE_URL: postgresql://langfuse:${DB_PASSWORD}@postgres:5432/langfuse
      NEXTAUTH_SECRET: ${NEXTAUTH_SECRET}
      NEXTAUTH_URL: https://langfuse.yourdomain.com
      # Optional: Email configuration
      # SMTP_CONNECTION_URL: smtp://user:pass@smtp.sendgrid.net:587
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped
    volumes:
      - ./logs:/app/logs
    networks:
      - langfuse-network

  postgres:
    image: postgres:15-alpine
    container_name: langfuse-postgres
    environment:
      POSTGRES_USER: langfuse
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: langfuse
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./backups:/backups
    ports:
      - "5432:5432"
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U langfuse"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - langfuse-network

networks:
  langfuse-network:
    driver: bridge

volumes:
  postgres_data:
EOF

# Create .env file
cat > .env << EOF
NEXTAUTH_SECRET=$NEXTAUTH_SECRET
DB_PASSWORD=$DB_PASSWORD
EOF

# Start
docker-compose up -d
docker-compose logs -f langfuse
```

## With HTTPS & Reverse Proxy

### Nginx Configuration

```bash
# Install Nginx
sudo apt update && sudo apt install -y nginx certbot python3-certbot-nginx

# Create Nginx config
sudo tee /etc/nginx/sites-available/langfuse > /dev/null << 'EOF'
upstream langfuse {
    server 127.0.0.1:3000;
}

server {
    listen 80;
    server_name langfuse.yourdomain.com;
    
    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name langfuse.yourdomain.com;

    # SSL certificates (auto-generated by certbot)
    ssl_certificate /etc/letsencrypt/live/langfuse.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/langfuse.yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    location / {
        proxy_pass http://langfuse;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket support
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
EOF

# Enable site
sudo ln -s /etc/nginx/sites-available/langfuse /etc/nginx/sites-enabled/

# Get SSL certificate
sudo certbot certonly --nginx -d langfuse.yourdomain.com

# Test and restart
sudo nginx -t
sudo systemctl restart nginx

# Auto-renewal
sudo systemctl enable certbot.timer
```

## Kubernetes Deployment

### Helm Chart (Recommended)

```bash
# Add Langfuse Helm repository
helm repo add langfuse https://langfuse.com/helm
helm repo update

# Create namespace
kubectl create namespace langfuse

# Create values.yaml
cat > values.yaml << 'EOF'
image:
  tag: latest

ingress:
  enabled: true
  hosts:
    - host: langfuse.yourdomain.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: langfuse-tls
      hosts:
        - langfuse.yourdomain.com

postgresql:
  enabled: true
  auth:
    username: langfuse
    password: changeme
    database: langfuse
  primary:
    persistence:
      enabled: true
      size: 20Gi

nextauth:
  secret: your-secret-key-here
  url: https://langfuse.yourdomain.com
EOF

# Deploy
helm install langfuse langfuse/langfuse \
  -n langfuse \
  -f values.yaml

# Verify
kubectl get pods -n langfuse
```

### Manual Kubernetes Manifests

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: langfuse

---
apiVersion: v1
kind: Secret
metadata:
  name: langfuse-secrets
  namespace: langfuse
type: Opaque
stringData:
  db-password: your-db-password-here
  nextauth-secret: your-nextauth-secret-here

---
apiVersion: v1
kind: ConfigMap
metadata:
  name: langfuse-config
  namespace: langfuse
data:
  DATABASE_URL: "postgresql://langfuse:$(DB_PASSWORD)@postgres:5432/langfuse"
  NEXTAUTH_URL: "https://langfuse.yourdomain.com"

---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: langfuse
  namespace: langfuse
spec:
  replicas: 2
  selector:
    matchLabels:
      app: langfuse
  template:
    metadata:
      labels:
        app: langfuse
    spec:
      containers:
      - name: langfuse
        image: langfuse/langfuse:latest
        ports:
        - containerPort: 3000
        env:
        - name: NEXTAUTH_SECRET
          valueFrom:
            secretKeyRef:
              name: langfuse-secrets
              key: nextauth-secret
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: langfuse-secrets
              key: db-url
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
        livenessProbe:
          httpGet:
            path: /
            port: 3000
          initialDelaySeconds: 30
          periodSeconds: 10

---
apiVersion: v1
kind: Service
metadata:
  name: langfuse-service
  namespace: langfuse
spec:
  selector:
    app: langfuse
  ports:
  - protocol: TCP
    port: 80
    targetPort: 3000
  type: ClusterIP

---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: langfuse-ingress
  namespace: langfuse
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - langfuse.yourdomain.com
    secretName: langfuse-tls
  rules:
  - host: langfuse.yourdomain.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: langfuse-service
            port:
              number: 80
```

## Backup & Recovery

### Automated PostgreSQL Backups

```bash
#!/bin/bash
# /opt/langfuse/backup.sh

BACKUP_DIR="/opt/langfuse/backups"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

mkdir -p "$BACKUP_DIR"

# Backup database
docker-compose exec -T postgres pg_dump \
  -U langfuse \
  langfuse | gzip > "$BACKUP_DIR/langfuse_$DATE.sql.gz"

# Delete old backups
find "$BACKUP_DIR" -name "langfuse_*.sql.gz" -mtime +$RETENTION_DAYS -delete

# Upload to S3 (optional)
# aws s3 cp "$BACKUP_DIR/langfuse_$DATE.sql.gz" \
#   s3://your-bucket/langfuse-backups/

echo "Backup completed: $BACKUP_DIR/langfuse_$DATE.sql.gz"
```

### Restore from Backup

```bash
# Extract and restore
gunzip < backups/langfuse_20260327_120000.sql.gz | \
  docker-compose exec -T postgres psql -U langfuse

# Or restore while postgres is running
docker-compose exec postgres bash << 'EOF'
gunzip < /backups/langfuse_20260327_120000.sql.gz | \
  psql -U postgres langfuse
EOF
```

### Cron Job for Daily Backups

```bash
# Add to crontab
crontab -e

# Add line:
0 2 * * * /opt/langfuse/backup.sh >> /opt/langfuse/backup.log 2>&1
```

## GraphRAG Integration

### Point GraphRAG to Self-Hosted Instance

```bash
# Set environment variables
export LANGFUSE_HOST="https://langfuse.yourdomain.com"
export LANGFUSE_PUBLIC_KEY="pk_xxxxxxxxxxxxx"
export LANGFUSE_SECRET_KEY="sk_xxxxxxxxxxxxx"

# Or create .env file
cat > .env << 'EOF'
LANGFUSE_HOST=https://langfuse.yourdomain.com
LANGFUSE_PUBLIC_KEY=pk_xxxxxxxxxxxxx
LANGFUSE_SECRET_KEY=sk_xxxxxxxxxxxxx
EOF

source .env

# Start GraphRAG server
source .venv/bin/activate
python -m backend.graphrag.cli serve --port 8000
```

### Monitoring Traces from CLI

```python
import os
from backend.graphrag.tracing import get_tracing_manager

# Set to your self-hosted instance
os.environ["LANGFUSE_HOST"] = "https://langfuse.yourdomain.com"

tracer = get_tracing_manager()
print(f"Connected to: {tracer.langfuse.base_url if tracer.langfuse else 'Disabled'}")
print(f"Tracing enabled: {tracer.enabled}")
```

## Security Best Practices

1. **Change Default Credentials**
   ```bash
   NEXTAUTH_SECRET=$(openssl rand -base64 32)
   DB_PASSWORD=$(openssl rand -base64 16)
   ```

2. **Use HTTPS Only**
   - Set up SSL certificates (Let's Encrypt)
   - Force HTTPS redirect
   - Use strong TLS ciphers

3. **Network Isolation**
   - Keep database service internal
   - Use Docker networks
   - Restrict firewall rules

4. **Regular Backups**
   - Daily automated backups
   - Test restore procedures
   - Store backups separately

5. **Update Regularly**
   ```bash
   docker pull langfuse/langfuse:latest
   docker-compose up -d
   ```
